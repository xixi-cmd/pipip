如何选择一门编程语言？

编程语言
随笔
发布日期:   2021-08-03
文章字数:   1.6k

最近想学一个新语言，但是难以选择，是选择前端还是后端？前端学什么，后端学什么？选择语言的时候总是会这样：

  1. 哇，那个JAVA好高大上！
  2. 哇，别人说Python好简单。
  3. 哇，C语言是计算机底层语言！好厉害！
  4. 哇，我要学汇编，电视里的黑客都用这个好像！

诸如此类，很多奇奇怪怪的想法。但是嘛，选择很重要，选择一门好的语言往往可以决定你走的路是平坦还是坎坷。语言这种东西，可以学一个再学一个，但是许多的人都是零基础入坑的，很显然第一门语言很难去选择，选了C，学到指针头炸裂，选了Python，类库多到你不知道选哪个来用，选了汇编，你发现16进制真的不是一般人可以看懂的，所以选择一门好的语言是一件重要的事。

后端

先来说说后端好了，毕竟它是很重要的。后端的语言多到你无法想，我就先说几个火的好吧。

Java

Java，1995年生，如今26岁，甲骨文公司的扛把子，老爸是高斯林，当然这都不重要。Java是一门可以编写跨平台应用软件的面向对象的程序设计语言，Android的应用就常常用Java来编写，如果你要做移动端的软件开发，那么Java是你的不二之选。

Python

Python，其实比Java要大，30年前出现的，所以她30岁了，他的老爸是荷兰程序员Guido van Rossum，江湖人称龟叔，龟叔作Python的目的出于对当时其他的编程语言的鄙视，这也都不重要。Python是一种通用语言，用于从服务器自动化到数据科学。Python对初学者来说是一门伟大的语言，因为它很容易阅读和理解。你也可以用Python做这么多事情，很容易坚持使用语言相当一段时间，在需要别的东西。当然Python也可以创建Web应用程序，如Instagram和帮助研究人员了解他们的数据。

C/C++

作为最古老的编程语言之一，C依然高居榜首，这归功于其可移植性以及微软、Oracle和苹果等科技巨头采用它。它与几乎所有系统兼容，很适合操作系统和嵌入式系统。

C#

Microsoft的宠儿，也是很多游戏开发者的宠儿，是微软的幼龙，C#是一种现代的面向对象编程语言，与当时商业软件开发人员广泛使用的Java相抗衡。它专为在微软平台上开发应用程序而设计，需要Windows上的.NET框架才能工作。与前一年一样，C#保持稳定的位置，名次没有重大变化。可以使用C#开发几乎所有应用程序，但它尤其擅长于Windows桌面应用程序和游戏开发。

php


php可以应用在 TCP/UDP服务、高性能Web、WebSocket服务、物联网、实时通讯、游戏、微服务等非 Web 领域的系统研发。
因为php是开源的，所以，很难找到她的爹，有问题吗，没有问题，

特点

php的特点是什么？

  1. 免费的，开源的，不只是她本身开源，和她有关的所有软件都是开免费的。
  2. 还有个什么，跨平台！每个平台都有对应的php解释器版本，指针对不同平台均编译出目标平台的二进制码(PHP解释器)，php开发的程序可以不经修改运行在windows、linux、unix等多个操作系统上。
  3. 占用内存非常少

还不明白吗？后端语言我只认php！

前端

说完后端了，来讲讲前端吧。什么，你不知道前端是什么？

前端，又称Web前端。是指Web网站的前台页面，在我们打开一个网站所看到网页界面的内容以及交互体验，一般都是由前端工程师进行开发设计的页面，该内容属于前端部分

HTML/CSS

HTML

先来说说html，她不是编程语言，她是超文本标记语言（HyperText Markup Language）也是她的全称。

CSS

CSS是基于html的，它的作用只是让你的界面变得更加的好看，他们仅仅用于浏览器。

JS（JavaScript）

JavaScript是许多人的第一个完整的编程语言。为什么？这是学习HTML和CSS后的逻辑下一步。JavaScript提供的行为一网站的部分。例如，当您看到表单字段指示错误时，这可能是JavaScript在工作。

JavaScript已经变得越来越受欢迎，它不仅仅出现在网络浏览器中。学习JavaScript将使你处在一个更好的地位，因为它是一个更通用、流行的语言，JavaScript现在似乎无处不在。

ajax（Asynchronous Javascript And XML）

说实话其实我也不是很懂这个，很少听到这个ajax，是在 2005年被Jesse James Garrett提出的新术语，用来描述一种使用现有技术集合的‘新’方法，包括: HTML 或 XHTML, CSS, JavaScript, DOM, XML, XSLT, 以及最重要的XMLHttpRequest。

它可能破坏浏览器的后退与加入收藏书签功能。在动态更新页面的情况下，用户无法回到前一个页面状态，这是因为浏览器仅能记下历史记录中的静态页面。一个被完整读入的页面与一个已经被动态修改过的页面之间的可能差别非常微妙；用户通常都希望单击后退按钮，就能够取消他们的前一次操作，但是在Ajax应用程序中，却无法这样做。不过开发者已想出了种种办法来解决这个问题，HTML5之前的方法大多是在用户单击后退按钮访问历史记录时，通过创建或使用一个隐藏的IFRAME来重现页面上的变更。（例如，当用户在Google Maps中单击后退时，它在一个隐藏的IFRAME中进行搜索，然后将搜索结果反映到Ajax元素上，以便将应用程序状态恢复到当时的状态）。

所以我不会推荐这个。
